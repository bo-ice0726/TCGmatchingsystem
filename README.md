# TCGマッチングシステム - セットアップガイド

複数デバイス・ブラウザ対応の大会管理アプリケーションです。

フロント（Vercel）とバックエンド（Render）で分離公開できる構成になっています。

## 構成

```
project/
├── server.js          # Node.js Express バックエンド
├── tournaments.json   # 大会データ
├── public/            # フロントエンドファイル
│   ├── index.html
│   ├── organizer.html
│   ├── participant.html
│   ├── app.js         # 共通API呼び出しロジック
│   ├── organizer.js
│   └── participant.js
└── package.json
```

## システム要件

- **Node.js** 12.0.0 以上
- **npm** またはその他のパッケージマネージャー
- **モダンブラウザ** (Chrome, Firefox, Safari, Edge 等)

## インストール手順

### 1. 依存パッケージのインストール

```bash
npm install
```

このコマンドで `express` と `cors` がインストールされます。

### 2. ローカルテスト

#### サーバーの起動

```bash
npm start
```

または

```bash
node server.js
```

ターミナルに以下のメッセージが表示されたら成功です：

```
TCGマッチングシステム サーバー起動: http://localhost:3000
ブラウザで http://localhost:3000 を開いてください
```

#### ブラウザでアクセス

```
http://localhost:3000
```

ローカル開発時は API の自動ローカルホスト接続に対応しています。

## デプロイメント

### バックエンド（Render）の公開

1. **Render にログイン**
   - https://render.com にアクセス

2. **新規 Web Service を作成**
   - リポジトリを接続
   - Runtime: Node.js
   - Build command: `npm install`
   - Start command: `node server.js`

3. **環境変数の設定（オプション）**
   - 必要に応じて PORT などを設定

4. **デプロイ**
   - Render が自動的にデプロイ
   - 公開 URL を取得（例：`https://example.onrender.com`）

### フロントエンド（Vercel）の公開

#### 方法1: public フォルダのみを Vercel にデプロイ

1. **Vercel にログイン**
   - https://vercel.com にアクセス

2. **`public` フォルダを新規プロジェクトとして作成**
   - または手動で `public` フォルダの内容をアップロード

3. **API_URL を Render の URL に変更**
   - [API_URL の設定](#api_urlの設定) を参照

#### 方法2: 同じリポジトリから分割デプロイ

- Vercel 側で Root Directory を `public` に設定
- `package.json` は不要（静的ファイルのみ）

### API_URL の設定

#### ローカル開発時

`public/app.js` の API_URL（現在は自動で localhost を使用）

#### Render デプロイ時

`public/app.js` の API_URL を変更：

```javascript
// Render のバックエンド URL に変更
const API_URL = 'https://your-render-app.onrender.com/api';
```

**重要**: Render と Vercel にデプロイしている場合
- Render のバックエンドが CORS を許可していることを確認
- Vercel のフロント URL が Render の CORS ホワイトリストに含まれていることを確認

## 使い方

### 開催者の手順

1. **大会を開く**
   - 大会名を入力
   - 対戦形式（トーナメント/スイスドロー）を選択
   - 「大会を開く」ボタンをクリック
   - 大会コード（例：ABCD-1234）が発行される

2. **開催者画面へ移動**
   - 発行されたコードから開催者画面へ
   - 参加者の参加を待つ

3. **大会開始**
   - 参加者が揃ったら「大会開始」ボタンをクリック
   - 自動的にマッチングが生成される

4. **ラウンド管理**
   - 全試合の結果が承認されたら「次のラウンドへ進む」をクリック
   - 全勝者が1人になると「大会を終了」ボタンが出現

### 参加者の手順

1. **大会参加ページへアクセス**
   - ホーム画面の「大会に参加」セクション
   - 開催者から発行された大会コードを入力
   - 「参加する」をクリック

2. **参加確定**
   - 名前を入力
   - 「参加する」をクリック

3. **マッチング確認**
   - 開催者が大会を開始するまで待機
   - 開始後、自動的にマッチング情報が表示

4. **試合実施**
   - 対戦相手と試合を実施
   - 結果を登録（勝者が登録ボタンをクリック）
   - 敗者が結果を承認

## マルチデバイス対応

- **スマートフォン**: フルレスポンシブ対応
- **タブレット**: 最適化されたレイアウト
- **PC**: デスクトップ向けUI

異なるデバイスから同じ大会コードでアクセス可能です。

## トラブルシューティング

### サーバーが起動しない

```bash
# Node.jsがインストールされているか確認
node --version

# npm モジュールが正しくインストールされているか確認
npm list
```

### 大会が見つからない

- 大会コードが正しく入力されているか確認
- サーバーが起動しているか確認
- 異なるブラウザでアクセスしてみてください

### リアルタイム更新が遅い

- ネットワーク接続を確認
- ブラウザのキャッシュをクリア
- ページをリロード

ブラウザのコンソール（F12）でエラー内容を確認してください。
