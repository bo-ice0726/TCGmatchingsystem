# TCGマッチングシステム - セットアップガイド

複数デバイス・ブラウザ対応の大会管理アプリケーションです。

## システム要件

- **Node.js** 12.0.0 以上
- **npm** またはその他のパッケージマネージャー
- **モダンブラウザ** (Chrome, Firefox, Safari, Edge 等)

## インストール手順

### 1. 依存パッケージのインストール

```bash
npm install
```

このコマンドで `express` と `cors` がインストールされます。

### 2. サーバーの起動

```bash
npm start
```

または

```bash
node server.js
```

ターミナルに以下のメッセージが表示されたら成功です：

```
TCGマッチングシステム サーバー起動: http://localhost:3000
ブラウザで http://localhost:3000 を開いてください
```

### 3. ブラウザでアクセス

ブラウザで以下のURLにアクセスしてください：

```
http://localhost:3000
```

## 使い方

### 開催者の手順

1. **大会を開く**
   - 大会名を入力
   - 対戦形式（トーナメント/スイスドロー）を選択
   - 「大会を開く」ボタンをクリック
   - 大会コード（例：ABCD-1234）が発行される

2. **開催者画面へ移動**
   - 発行されたコードから開催者画面へ
   - 参加者の参加を待つ

3. **大会開始**
   - 参加者が揃ったら「大会開始」ボタンをクリック
   - 自動的にマッチングが生成される

4. **ラウンド管理**
   - 全試合の結果が承認されたら「次のラウンドへ進む」をクリック
   - 全勝者が1人になると「大会を終了」ボタンが出現

### 参加者の手順

1. **大会参加ページへアクセス**
   - ホーム画面の「大会に参加」セクション
   - 開催者から発行された大会コードを入力
   - 「参加する」をクリック

2. **参加確定**
   - 名前を入力
   - 「参加する」をクリック

3. **マッチング確認**
   - 開催者が大会を開始するまで待機
   - 開始後、自動的にマッチング情報が表示

4. **試合実施**
   - 対戦相手と試合を実施
   - 結果を登録（勝者が登録ボタンをクリック）
   - 敗者が結果を承認

## マルチデバイス対応

- **スマートフォン**: フルレスポンシブ対応
- **タブレット**: 最適化されたレイアウト
- **PC**: デスクトップ向けUI

異なるデバイスから同じ大会コードでアクセス可能です。

## トラブルシューティング

### サーバーが起動しない

```bash
# Node.jsがインストールされているか確認
node --version

# npm モジュールが正しくインストールされているか確認
npm list
```

### 大会が見つからない

- 大会コードが正しく入力されているか確認
- サーバーが起動しているか確認
- 異なるブラウザでアクセスしてみてください

### リアルタイム更新が遅い

- ネットワーク接続を確認
- ブラウザのキャッシュをクリア
- ページをリロード


## 公開方法

このアプリは、フロントエンドを Vercel、バックエンドを Render で公開する構成を想定しています。

### フロントエンド

- `public/` フォルダ内の `index.html` / `organizer.html` / `participant.html` / `app.js` などを Vercel にデプロイします。
- フロントエンドは Vercel のホスト上で動作し、RenderのAPIを呼び出します。

### バックエンド

- `server.js` を Render で動かします。
- `server.js` は `public/` フォルダを静的配信し、APIルートを提供します。

### APIのURL設定

フロントエンドの `public/app.js` で `API_URL` を更新してください。

```js
const API_URL = 'https://your-app.onrender.com/api';
```

### Renderでの基本設定

- ビルドコマンド: `npm install`
- 起動コマンド: `npm start`
- ポート: 自動で `PORT` 環境変数が割り当てられます

### Vercelでの基本設定

- GitHubリポジトリをVercelに接続
- `public/` を静的サイトとして公開
- `public/app.js` の `API_URL` を Render の URL に変更

### 例

- Renderバックエンド: `https://your-app.onrender.com`
- Vercelフロントエンド: `https://your-app.vercel.app`

## もっと簡単に公開するなら

RenderやRailwayなら、GitHubリポジトリを接続して自動デプロイを有効にするだけで公開できます。
